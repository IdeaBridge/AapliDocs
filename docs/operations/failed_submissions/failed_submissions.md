## Failed Submissions

To access the Failed Submissions page, click on the widget labelled "Failed Submissions" on the bottom row of the Operations tab group on the admin home page.

{% include image.html file="failsubwidget.png" class="docimage" url="https://admin-dev.aapli.app/" alt="Failed Submissions Widget" caption="Failed Submissions Widget" %}

This table displays all of the submissions from applications in the suborganisation that didn't succeed, and provides the user with the opportunity to either re-submit their log if need be, or delete it.

{% include image.html file="failsubs.png" class="docimage" url="https://admin-dev.aapli.app/admin/operations/failed-approval" alt="Failed Submissions" caption="Failed Submissions" %}

1. **Resubmit:** If the user wishes to try submitting their input again through this table, they can click this button and the concerned application will attempt sending it again.
2. **Id:** The unique ID of the attempted submission.
3. **Title:** The title of the attempted submission.
4. **Application Name:** The name of the application responsible for the failed submission.
5. **Created Date:** The date of the creation of the submission.
6. **Actions:** The user can choose to the failed submission if there is no need to attempt it anymore.
